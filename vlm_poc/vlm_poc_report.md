## 1. 核心發現：Callout 標注在 PDF 圖形層，不在文字層

pdfplumber/fitz 從 Construction Callout 頁面提取的文字只有頁首和頁碼，**零個 zone keyword**。所有做工標注（zone 名稱 + 術語）都嵌在 PDF 的圖形層（image/annotation layer），不是 text layer。

**驗證範圍**：80+ PDFs，橫跨 Knit/Woven × TOP/BOTTOM/OUTERWEAR/LEGGINGS

**結論**：text-based pipeline 從 Callout 頁面提取的 ISO 數據**全部來自非 Callout 頁面**（Measurement Chart、BOM、metadata），存在系統性汙染。

---

## 2. AH=301 汙染根因確認

**問題**：v4.3 aggregate 顯示 WOMENS TOP AH=301 佔 84%，但 knit armhole 應該是 504/514 overlock 或 406 coverstitch。

**根因**：Measurement Chart 頁面的 POM 名稱 "ARMHOLE STRAIGHT" 後接尺寸 "30 1/4"，text pipeline fuzzy regex 將 `ARMHOLESTRAIGHT...301⁄` 誤讀為 ARMHOLE + ISO 301。

**VLM 修正**：D80999（Knit TOP）的 Callout 頁面清楚標示：
> ARMHOLES: 1/4" 2N STRADDLE STITCH → **ISO 406** (coverstitch)

這是 VLM 與 v4.3 aggregate 最具決策意義的差異。

**DISAGREE 分析**：

**Type A — VLM 修正了 AGG 汙染（1 筆）**：
- D80999.AH: VLM=406 vs AGG=301 → MC 尺寸值汙染

**Type B — AGG 記錄的是 seam join，VLM 記錄的是 topstitch finish（9 筆）**：
- SH/SA 多個設計：AGG=401(chainstitch join) vs VLM=301(SNTS topstitch)

**Type C — AGG 小樣本可能不準（4 筆）**：

- PK AGG=501(72%) vs VLM=301(SNES)，pocket 的 text pipeline 結果可疑

## 4. GAP/ONY 術語 → ISO 對照（Glossary Mapping）

Pipeline 使用 longest-match-first 策略，已建立 100+ 條 Glossary 對照：

**關鍵區分**：
- `2N STRADDLE STITCH` = **406** (coverstitch) ← 正確
- `2NTS` = 2 Needle Top Stitch = dual **301/401** ← 注意！不是 406
- `SNTS` = Single Needle Top Stitch = **301**
- `SNES` = Single Needle Edge Stitch = **301**
- `FLATLOCK` = **607** (併縫)
- `PEARL MERROW` = overlock 變體 ≈ **504/514**

**待補充**：
- `PEARL MERROW` 尚未加入 Glossary（D75816 用到）
- `C/S TAPING` (coverstitch taping) = **406**，需確認
- `DNTS` (double needle topstitch) — 歧義：可能是 301×2 或 406

## 5. Pipeline 架構

```
PDF → detect_callout_pages()     Word count < 40 = image-based
    → render_callout_pages()     3x zoom = 216 DPI PNG
    → [VLM reads image]          Claude Vision / GPT-4V / future API
    → map_terminology_to_iso()   Longest-match Glossary lookup
    → map_zone_to_l1()           English zone → 38-code L1
    → process_vlm_output()       Handle "ALL SEAMS" defaults
    → vlm_callout_extracts.json  Per-design zone-ISO mapping
```

**PoC 模式**：Claude 手動讀圖 → 填 vlm_raw_extracts.json → 腳本做 Glossary mapping  
**未來 API 模式**：Claude Vision API 自動讀圖 → 全自動
