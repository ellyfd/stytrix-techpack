# L2 混淆對矩陣 — Hard Negative Pairs for AI Training

> **用途**：Contrastive learning 的 hard negative pairs，讓 AI 訓練時特別學會區分這些「長得像但不同」的 L2 對
> **來源**：L2_Visual_Differentiation_FullAnalysis_修正版.md（282 個 L2 image-based 視覺描述）
> **建立日期**：2026-04-20
> **分級標準**：
> - 🔴 **外觀完全相同**：sketch 上完全看不出差異，只能靠 callout 文字 → needs_text=true
> - 🟡 **需看細節**：有差異但不明顯，需指定看哪個特徵 → 給 AI 特徵指引
> - 🟢 **容易混淆但有差異**：結構相似但有可辨識的視覺線索 → decision tree 可解

## 五、訓練建議

### Hard Negative Mining 策略

1. **🔴 對（19 對）**：這些 pair 不用做視覺對比訓練——AI 學不會。改為：
   - 訓練 AI 判到「最近的可判 L2」→ 輸出 merged_candidates 清單
   - 文字通道（OCR callout）做最後選擇

2. **🟡 對（16 對）**：這些是 contrastive learning 的主戰場：
   - 每對製作 positive/negative 圖片 pair
   - prompt 裡明確標示「看什麼特徵」
   - 例：PK_004 vs PK_005 → crop 袋口區域 → 齒紋有/無

3. **🟢 對（14 對）**：這些是 VLM decision tree 就能解的：
   - 不需要專門的 contrastive 訓練
   - 在 prompt 裡多加一層判定節點即可
   - 例：SB 短褲 vs 長褲 → 先看褲長再看剪接線形狀
