# StyTrix Techpack 網站架構圖

範圍：線上網站（前端 + Vercel Functions + GitHub Actions pipeline + 外部服務 + 資料相依）的全貌。
只畫「跑在線上或會被線上 trigger 到」的東西；純線下一次性的 `scripts/` 規則產線不在本圖內（細節見 `pom_rules_pipeline_guide_v2.md`）。

---

## 1. 高階流程（Browser → Vercel → 外部 API / Actions）

```mermaid
flowchart LR
  U([使用者瀏覽器])
  Admin([Admin 使用者<br/>需 x-admin-token])

  subgraph Vercel["Vercel（stytrix-techpack.vercel.app）"]
    direction TB
    SS["Static 託管<br/>index.html · LOGO.png<br/>data/ (含 l1_*_v1.json、bucket_taxonomy)<br/>pom_rules/ · l2_l3_ie/ · recipes/<br/>General Model_Path2/"]
    F1["api/analyze.js<br/>(Node, maxDuration 300s)"]
    F2["api/push-pom-dict.js<br/>(Node, maxDuration 30s)"]
    F3["api/ingest_token.js<br/>(Node, maxDuration 10s)"]
  end

  CV[(Anthropic<br/>Claude Vision API)]
  GH[(GitHub Contents API<br/>ellyfd/stytrix-techpack)]

  subgraph Actions["GitHub Actions"]
    A1["rebuild_master.yml<br/>push 到 data/ingest/uploads/**<br/>觸發 recipes_master rebuild"]
    A2["build_l2_l3_ie.yml<br/>push 到 五階層展開項目_*.xlsx<br/>觸發 l2_l3_ie rebuild"]
  end

  U -- "GET /<br/>載入 SPA + JSON 資料" --> SS
  U -- "POST /api/analyze<br/>(image + context + mode)" --> F1

  Admin -- "POST /api/push-pom-dict<br/>翻譯表編輯器" --> F2
  Admin -- "POST /api/ingest_token<br/>取 GitHub PAT" --> F3
  Admin -. "直連 GitHub（PDF/PPTX/xlsx 上傳、JSON patch）<br/>用拿到的 PAT 直接 PUT contents" .-> GH

  F1 -- "messages.create<br/>Pass 1 + Pass 2" --> CV
  F2 -- "PUT data/pom_dictionary.json" --> GH

  GH -. "push 事件觸發" .-> A1
  GH -. "push 事件觸發" .-> A2
  A1 -- "commit rebuilt data/" --> GH
  A2 -- "commit rebuilt l2_l3_ie/" --> GH
  GH -. "[skip ci] commit 觸發<br/>Vercel redeploy" .-> Vercel
```

**關鍵設計**：
- 純靜態 SPA（無 bundler、無 `package.json`），React 走 CDN，整個 app 在 `index.html` 內聯。
- 所有 functions 都是 Vercel **Node.js runtime**，不是 Edge runtime。Anthropic 模型為 `claude-sonnet-4-6`。
- `api/analyze.js` 在 module init 階段 `fs.readFileSync` 讀四份：`data/l2_visual_guide.json`、`data/l2_decision_trees.json`、`data/l1_standard_38.json`、`techpack-translation-style-guide.md`。靠 `vercel.json` 的 `includeFiles: "{data/**,techpack-translation-style-guide.md}"` 把檔打包進 function bundle（先前改成 fetch 會 hang 到 timeout）。
- `api/ingest_token` 的作用只有一個：把 `GITHUB_PAT` 給已驗證的 admin，讓瀏覽器**直連** GitHub API，繞過 Vercel 4.5MB body 上限（PPTX 常超過，聚陽 xlsx 固定 9.5MB）。上傳路徑統一走這條（2026-04-24 移除原本規劃但未實作的 `api/ingest_upload` 小檔路徑）。
- **Pipeline 外送包 / 回流**（PackageModal + ResultsUploadModal）**永遠**走直連 GitHub 不經 Vercel endpoint：下載用 raw.githubusercontent.com 拉檔、瀏覽器 JSZip 組包；回流用 GET + PUT `/contents/<path>`。
- 資料重建**不經過 Vercel function**——是 GitHub Actions workflow 在 GitHub 端跑的。

## 4. Ingest Pipeline（PATH2 Phase 2：PDF/PPTX → 自動重建 recipes_master）

使用者丟一份新 Techpack 進 UI，到 `data/recipes_master.json` 更新、Vercel 重新部署，**完全不需要本機環境**。

```mermaid
flowchart TB
  Uploader([Admin 在管理選單<br/>📤 上傳 Techpack])

  subgraph BrowserPath["上傳路徑（無分流，統一直連 GitHub）"]
    direction LR
    BP2["所有檔案<br/>→ /api/ingest_token 取 PAT<br/>→ 瀏覽器直連 GitHub API<br/>(2026-04-24 前曾規劃 ≤4.5MB 走 /api/ingest_upload,已移除)"]
  end

  GH1[("GitHub repo<br/>data/ingest/uploads/&lt;file&gt;")]

  subgraph WF["GitHub Actions：rebuild_master.yml<br/>trigger: push paths data/ingest/uploads/**<br/>(workflow_dispatch 也支援，可 force 重跑)"]
    direction TB
    S1["Step 1 extract_raw_text.py<br/>PDF 首頁元資料 + D-number;PPTX 逐頁文字;<br/>callout 頁面評分後 216 DPI 渲染 PNG<br/>→ metadata/designs.jsonl (append)<br/>→ pptx/*.txt<br/>→ pdf/callout_images/*.png<br/>→ pdf/callout_manifest.jsonl<br/>支援 --allow-empty / --force"]
    S2b["Step 2b vlm_pipeline.py --map-iso<br/>(順序上 2b 先 2a 後,確保 2a 吃到新 VLM facts)<br/>若 vlm_raw_extracts.json 不存在 + ANTHROPIC_API_KEY 有設:<br/>直接呼叫 Claude Haiku 分析 callout PNG<br/>→ 200+ ISO 術語 + 30+ zone 對映 + bucket 欄位<br/>→ data/ingest/<b>vlm/</b>{facts.jsonl, vlm_callout_extracts.json}<br/>continue-on-error: true"]
    SNotice["Step 2b failure notice<br/>if steps.vlm.outcome == 'failure':<br/>寫警告到 $GITHUB_STEP_SUMMARY<br/>設 VLM_FAILED=true"]
    S2a["Step 2a extract_unified.py<br/>全量合併 4 來源 (PPTX 中文 / PDF 英文 / cb / dir5)<br/>PPTX JSON > PPTX txt > 其他;同時 merge 2b 產的 VLM facts<br/>→ data/ingest/unified/{dim,facts}.jsonl"]
    S3["Step 3 build_recipes_master.py --strict<br/>5 來源合併:<br/>• iso_lookup_factory_v4.3.json (primary, 230)<br/>• iso_lookup_factory_v4.json (fallback, 282)<br/>• data/construction_bridge_v6.json<br/>• recipes/*.json (72 檔)<br/>• data/ingest/consensus_v1/entries.jsonl (275 條)<br/>+ data/ingest/*/facts.jsonl(含 vlm/)<br/>L1 對 l1_standard_38、bucket 對 bucket_taxonomy(59)驗證<br/>→ data/recipes_master.json<br/>→ data/iso_dictionary.json<br/>→ data/l1_standard_38.json<br/>違規 exit 1"]
    S4["git rm data/ingest/uploads/*<br/>+ git add 重建結果 + data/ingest/vlm/ (persist)<br/>若 VLM_FAILED=true,commit msg body 加 ⚠ VLM failed<br/>commit msg: 'chore(data): auto-rebuild recipes_master [skip ci]'"]
    S1 --> S2b --> SNotice --> S2a --> S3 --> S4
  end

  GH2[("GitHub main<br/>data/recipes_master.json 等")]
  V1["Vercel 重新部署<br/>前端吃到新 recipes"]

  Uploader --> BrowserPath
  BP2 --> GH1
  GH1 -- "push 事件" --> WF
  S4 -- "git push origin HEAD" --> GH2
  GH2 -. "push webhook" .-> V1
```

**細節**：
- workflow 只在 `data/ingest/uploads/**` 有變動時觸發；手動 `workflow_dispatch` 可帶 `force=true` 忽略已處理設計全部重跑。
- Step 1 PDF 掃首頁抓季節 / 品牌 / 設計類型、用正規式抽 D-number；callout 頁面評分系統（ISO +3、標題 +3、縫線 +2），硬排除等級評估 / 靈感 / 狀態表格。
- **Step 2b 排在 Step 2a 之前**（2026-04-23 P1 修正，commit `32dc78c`），讓 Step 2a 的 unified 合併能吃到本次 run 剛產出的 VLM facts，而不是上一輪的舊資料。
- **Step 2b 現在自動呼 Claude**：若沒有預填的 `vlm_raw_extracts.json` 但 `ANTHROPIC_API_KEY` 有設，會對 Step 1 產出的每張 callout PNG 呼叫 Claude Haiku（`anthropic>=0.25.0`），輸出到 facts.jsonl（含 `bucket` 欄位，Step 3 才能用）。之前是 no-op——PNG 從沒真的送去分析過。`continue-on-error: true` 維持不變。
- Step 2a **全量重建**（吃所有歷史 facts + 新增的），不是增量 append，輸出 deterministically 一致。
- Step 3 aggregation 優先級：`same_sub (recipe) > same_bucket (consensus + facts) > same_gt (v4.3) > general (v4) > cross_design (bridge)`。
- Step 4 commit 會 `git add data/ingest/vlm/`，讓 VLM 分析結果持久化到 repo（之前沒存）。
- workflow commit 帶 `[skip ci]` 避免重建結果觸發自己；Vercel 偵測 push → 自動重新部署。
- **前端 CI 追蹤**：`UploadModal` 拿 sha 後 poll `GET /repos/.../actions/runs?head_sha=<sha>`，依 workflow 實際步驟順序顯示：`1｜拆解文字 & callout 圖片` → `2b｜VLM 分析 callout (PDF)` → `2a｜統一萃取 (PPTX)` → `3｜重建 recipes_master` → `提交資料`。  
  Actions API 回 403/404（私人 repo / token 缺 `actions:read` scope）時 **graceful fallback**：先試不帶 token 重試，仍失敗就直接顯示 GitHub Actions 連結不再 poll（前端不會卡死）。

## 5. Admin 通道總表

管理選單（▾）分兩組：

```
Pipeline 對外協作
  📦 下載 Pipeline 包
  📥 上傳 Pipeline 結果
  📤 上傳 Techpack
——
管理工具
  📝 POM 翻譯表編輯
  📚 權威手冊登記表
```

| 入口 | 做什麼 | 走哪條路 | 驗證 | Body 大小 |
|---|---|---|---|---|
| 📦 **下載 Pipeline 包**（PackageModal） | 組 zip 給外部協作單位在本機跑 pipeline — 含 4 個 py + ~84 個 reference JSON + 自動產 README / VERSION / `.env.example` / `run.sh` | 取 PAT → GitHub API 抓 main commit sha + recipes/ 列表 → 並行拉 raw.githubusercontent.com → JSZip 瀏覽器組 zip → 觸發下載 | `x-admin-token` → PAT | — |
| 📥 **上傳 Pipeline 結果**（ResultsUploadModal） | 吃外部送回的 zip/jsonl,merge 到 `data/ingest/{unified,vlm}/facts.jsonl` + `metadata/designs.jsonl` | JSZip 瀏覽器解壓 + 認 path → GET 現有 → merge(append / append-dedup by design_id) → 直連 `PUT contents` | 瀏覽器帶 `Authorization: token <PAT>` | 無實質上限 |
| 📤 **上傳 Techpack**（UploadModal） | 丟 PDF/PPTX 進 `data/ingest/uploads/`，觸發 Actions 重建 | `POST /api/ingest_token` → 瀏覽器直連 GitHub `PUT contents` | `x-admin-token` → 後續直連用 PAT | 無實質上限 |
| 📝 **POM 翻譯表編輯**（AdminModal） | 編 `data/pom_dictionary.json`，自動 diff + 產 commit message | `POST /api/push-pom-dict`（Vercel endpoint 代 commit） | `x-admin-token` | 小 |
| 🛠 **IE 底稿管理**（IEAdminModal） | 聚陽送新版 `五階層展開項目_*.xlsx` 上傳,觸發 `build_l2_l3_ie.yml` 重建 `l2_l3_ie/*.json` | `POST /api/ingest_token` → 瀏覽器直連 GitHub `PUT`(xlsx 9.5MB 超過 Vercel 4.5MB) | `x-admin-token` → 後續直連用 PAT | 無實質上限 |
| 📚 **權威手冊登記表**（ManualsModal） | 純 read-only 檢索，顯示 7 份權威手冊角色 / 引用鏈 | 無後端呼叫 | — | — |

> ~~🩹 上傳 Patch (JSON)（PatchUploadModal）~~ — 2026-04-23 移除,由 📥 上傳 Pipeline 結果取代(新通道吃 facts.jsonl 而非 recipes_master.json patch,更符合外部協作實際流程)。

所有「瀏覽器直連 GitHub」都共用 `/api/ingest_token` 發的 `GITHUB_PAT`（需 `contents: write`），PAT 只在 admin 的瀏覽器 session 內活著，不落地。

### ResultsUploadModal 認得的 target

| 路徑 | 合併模式 | 說明 |
|---|---|---|
| `data/ingest/unified/facts.jsonl` | append | 4 來源合併後的 facts |
| `data/ingest/vlm/facts.jsonl` | append | VLM 對 callout 的分析 |
| `data/ingest/metadata/designs.jsonl` | append-dedup by `design_id` | 設計元資料,避免重複 |

**環境變數**（Vercel Project Settings）：
- `ANTHROPIC_API_KEY` — `api/analyze.js` 呼叫 Claude Vision
- `ADMIN_TOKEN` — 所有 admin 端點的共享 token
- `GITHUB_PAT` — 寫回 GitHub 用的 PAT（contents: write）

GitHub Actions secrets（repo settings）：
- `ANTHROPIC_API_KEY` — Step 2b VLM pipeline 用
- `GITHUB_TOKEN`（自動注入）— Step 4 push 用
